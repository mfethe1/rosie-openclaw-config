## 1. oh-my-opencode v3.9.0 (Feb 26, 2026)

### Key Changes

**Worktree-Aware Planning**
- `/start-work --worktree` flag: Prometheus creates plans scoped to a specific worktree. Atlas picks up worktree path in boulder continuation.
- Impact: parallel feature branches without cross-contamination. Directly relevant to our multi-agent parallel dispatch pattern.

**Model Routing**
- Gemini 3 → 3.1 Pro across all categories.
- Kimi added to writing category fallback.
- GPT detection simplified to name-based matching (no more o-series edge cases).
- **Hephaestus auto-commit removed** — agents no longer commit without explicit request. Critical safety improvement.

**Reliability Hardening (all relevant to our architecture)**
- Ralph Loop: Completion detection scoped to messages since loop start. In-flight guard prevents infinite re-trigger. Race condition between session reset and TUI switch eliminated.
- Todo Continuation: Blocked todos excluded from incomplete count — prevents infinite loops on dependent tasks.
- Background Agent: Notification queue for idle sessions. Pending notifications cleaned on session delete.
- Token Limiter: Truncation respects newline boundaries.
- Compaction: Timeout + cleanup prevents indefinite hangs during rate limiting.

**Hashline Edit Benchmarks**
- 46-test benchmark suite for hashline-edit. Deduplication and validation fixes. Diff context limited to 3 lines (prevents oversized hunks).
- We should run equivalent benchmarks on our own edit path.

### Assessment vs. Our Architecture

| Dimension | oh-my-opencode v3.9.0 | Our System |
|---|---|---|
| **Deterministic Execution** | ✅ Worktree scoping, in-flight guards, Ralph completion detection | ✅ cron-bounded, step skeleton |
| **Verification Gates** | ✅ 3,391 tests, hashline benchmark suite | ✅ smoke_test.sh + eval-log.md |
| **Cost/Maintenance** | 🟡 187 files changed = high internal churn | 🟢 Stable, low churn |
| **Model Portability** | ✅ Gemini 3.1 Pro, Kimi, GPT all supported | ✅ 10+ models rotated |

### Recommendations
- **KEEP** monitoring. Reopen in active scope. v3.9.0 is a substantial quality signal.
- **TEST** worktree-aware planning (`--worktree` flag) for parallel feature branch use cases in our Prometheus/Atlas pipeline. P3.
- **TEST** hashline-edit benchmark approach — adapt 46-test structure for our own edit validation suite. P3.
- **KEEP** our Hephaestus equivalent auto-commit disabled (we were already aligned; oh-my-opencode just caught up).
- **STOP** treating oh-my-opencode as dead. Resume weekly tracking.
