## 1. oh-my-opencode (oh-my-openagent)
* **Deterministic Execution Guarantees:** Medium. Focuses on multi-model orchestration with background task management and specialized sub-agents (Explorer, Oracle, etc.), but execution flow depends on complex agent interactions.
* **Verification Gates:** High. Strong verification through LSP/AST tools integration and specialized reviewer agents.
* **Cost/Maintenance Burden:** High. Complex orchestration framework with many configuration files and sub-agent definitions (e.g., 14 overridable agents).
* **Model Portability:** High. Native multi-provider support (Anthropic, OpenAI, Google) with fallback priorities.
* **Recommendation: KEEP** monitoring as a baseline for complex orchestration, but avoid adopting due to heavy configuration overhead.

## 2. Antfarm
* **Deterministic Execution Guarantees:** High. Operates reliably and deterministically using crons, YAML, and SQLite for state management.
* **Verification Gates:** High. Ships with a specialized built-in team (planner, developer, verifier, tester, reviewer) working in repeatable workflows.
* **Cost/Maintenance Burden:** Low. "Batteries-included" with a single install command, tightly integrated directly with OpenClaw.
* **Model Portability:** High. Relies on OpenClaw's underlying model portability.
* **Recommendation: TEST**. The deterministic state-machine approach (SQLite + YAML) natively integrated with OpenClaw is highly relevant and warrants immediate testing.

## 3. Ralph-loop
* **Deterministic Execution Guarantees:** Low/Medium. A simple while-loop evaluating stopping criteria (iteration limits, token limits, cost).
* **Verification Gates:** Medium. Relies on continuous feedback injection from failed test/validations to guide the next loop iteration.
* **Cost/Maintenance Burden:** Low maintenance (very simple core architecture), but potentially High execution cost if loop boundaries are not strictly managed.
* **Model Portability:** High. Built on Vercel AI SDK 6, supporting full AI Gateway formats.
* **Recommendation: STOP** treating as a standalone framework. Instead, extract the specific conceptual pattern (the basic while-loop task executor) for small isolated tasks.
