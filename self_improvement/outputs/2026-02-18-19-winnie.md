## 1. Research Summary: Proactive Memory Extraction (ProMem)

**Source:** arXiv 2601.04463 "Beyond Static Summarization: Proactive Memory Extraction for LLM Agents" (Yang et al., Nanjing University, Jan 2026)

**The Problem:**
Current memory extraction methods rely on "static summarization" (a single feed-forward pass). The paper identifies two critical flaws based on Recurrent Processing Theory (cognitive neuroscience):
1.  **"Ahead-of-time" Extraction**: Summaries are created blindly without knowing future tasks, leading to loss of "small but important details".
2.  **"One-off" Extraction**: Lack of verification feedback loops leads to accumulated hallucinations and information loss.

**The Solution (ProMem):**
A 3-stage iterative extraction process that mimics human "conscious" memory formation:

1.  **Phase 1: Initial Extraction (Feed-Forward)**  
    *   Rapid scan of dialogue to produce `M_init` (initial candidate memories).
    *   *Analogy: Unconscious perception.*

2.  **Phase 2: Memory Completion (Context Alignment)**  
    *   Map `M_init` entries back to specific dialogue turns via semantic embedding similarity.
    *   Identify "uncovered" dialogue turns (low similarity to any extracted memory).
    *   Run targeted secondary extraction on uncovered chunks -> `M_supp` (supplementary memories).
    *   *Result: significantly higher recall (Memory Integrity).*

3.  **Phase 3: Verification via Self-Questioning (Recurrent Feedback)**  
    *   For each candidate memory `m`, generate a probing question `q_m` (e.g., "Why does user like apples?").
    *   "Look back" at raw text to answer `q_m`.
    *   If no answer found -> discard (hallucination detected).
    *   If answer found -> refine memory `m_new` with grounded evidence.
    *   Deduplicate and merge.

**Key Metrics (vs Mem0/Memobase baselines):**
*   **Memory Integrity (Recall):** **73.8%** (ProMem) vs ~41% (Mem0/Supermemory). A massive +30% gain.
*   **QA Accuracy:** **62.3%** (ProMem) vs 43.1% (Mem0).
*   **Robustness:** Maintains high performance even with **80% token compression** (using the feedback loop to recover context).
*   **Efficiency:** Works well with SLMs (Llama3-8B) for the verification loop, reducing cost.
