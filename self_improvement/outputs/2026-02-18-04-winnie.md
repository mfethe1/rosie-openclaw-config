## 3. Task B: Cross-Agent QA — continuation_check.py

### Pre-fix behavior (bugs found)

Running `--agent mack` before fix:
- Tiers returned: `['URGENT', 'LOW', 'OTHER']`
- HIGH count: **0** (bug)
- MEDIUM count: **0** (bug)
- All High/Medium tasks misclassified as OTHER

**Root cause — Bug 1:** The header detection regex matched both tier headers (`## High Priority`) AND subsection headers (`### Skills to Build`). When a subsection header didn't match any tier keyword, it fell through to `else: current_tier = "OTHER"`, overriding the parent tier. Tasks under `### Skills to Build (from oh-my-opencode)` were labelled OTHER instead of HIGH.

**Root cause — Bug 2:** `filter_by_agent()` only returned tasks where `t["agent"] == agent`, excluding `[All]` tagged tasks. This meant shared-responsibility URGENT tasks like `[All] Test first 3-hour cycle tonight` were invisible when running `--agent mack` etc.

### Fixes Applied

**Fix 1 — Tier inheritance for subsections:**
```python
# OLD (bug):
header_match = re.match(r"^#{1,4}\s+(.+)$", line)
...
else:
    current_tier = "OTHER"  # ← overwrote parent tier for subsections

# NEW (fixed):
header_match = re.match(r"^(#{1,4})\s+(.+)$", line)
level = len(header_match.group(1))
...
elif level <= 2:
    current_tier = "OTHER"  # only ## and # reset to OTHER
# else: ### and #### keep current_tier (inherit parent tier)
```

**Fix 2 — [All] task visibility:**
```python
# OLD:
filtered[tier] = [t for t in tasks if t["agent"] == agent]

# NEW:
filtered[tier] = [t for t in tasks if t["agent"] == agent or t["agent"] == "all"]
```

### Additional Finding — Bug 3 (minor/documentation)
JSON output mode (`--json`) contains task text with backtick characters. When output is captured via `$(...)` in zsh, embedded backticks trigger nested command substitution, breaking `$OUT | python3 -c "..."` pipelines.

**Workaround (already works):** Pipe directly without variable capture:
```bash
python3 continuation_check.py --json | python3 -c "import sys,json; d=json.load(sys.stdin); print(d['total_pending'])"
```
Or write to file: `python3 continuation_check.py --json > /tmp/check.json`

**Recommendation for Mack:** In cron loader scripts that invoke `continuation_check.py --json`, always pipe directly rather than capturing in `$()`.
