## Task 2: SimpleMem Stage 1 Compression Quality Validation

### Method
Stored a 483-char test entry via the local memU bridge (localhost:8711) and inspected the
`compression` metadata field. Also read the full `compress_stage1()` implementation in
`memu_server/server.py`.

### Finding: Compression is NOT Active 🔴

```
compression.generated = False
compression.provider  = None
compressed_content    = original (1.00 ratio)
auto_tags             = []
```

### Root Cause
`memu_server/start.sh` does **not** source `~/.openclaw/secrets/deploy.env`, so
`ANTHROPIC_API_KEY` is absent from the memu server's process environment.

`compress_stage1()` explicitly checks:
```python
anthropic_key = os.environ.get("ANTHROPIC_API_KEY") or os.environ.get("CLAUDE_API_KEY")
if not anthropic_key and not openai_key:
    return text, False, None   # ← THIS IS THE PATH WE'RE HITTING
```

### Fix Required (Mack — B-015)
Add to `memu_server/start.sh`, before the server is launched:
```bash
# Load API keys for LLM-powered features (SimpleMem compression, auto-tags)
if [ -f "$HOME/.openclaw/secrets/deploy.env" ]; then
    set +u   # allow unbound vars during source
    source "$HOME/.openclaw/secrets/deploy.env"
    set -u
fi
```

### Expected impact after fix
- `compression.generated = True` for entries >300 chars
- `compression.provider = anthropic`
- `auto_tags` populated by `extract_auto_tags(compressed_content)`
- memU search quality improves (searches over compressed summaries not raw text)
