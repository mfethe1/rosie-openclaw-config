## 1. What Is EvoAgentX?

EvoAgentX is an open-source framework (EMNLP 2025) for **automated construction, execution, and evolutionary optimization** of multi-agent workflows.

**Architecture — 5 Layers:**
1. Basic components (LLM, tools, memory primitives)
2. Agent (specialized roles)
3. Workflow (graph-based multi-agent orchestration)
4. **Evolving** — self-improvement engine (3 algorithms: TextGrad, AFlow, MIPRO)
5. **Evaluation** — built-in scorer per task/metric

**Evolution Algorithms:**
| Algorithm | What it optimizes | Mechanism |
|---|---|---|
| **TextGrad** | Agent prompts (role/task descriptions) | Text-based "gradient": treat LLM feedback as loss signal, refine prompt iteratively |
| **AFlow** | Workflow topology | Search over task decomposition / execution graph variants |
| **MIPRO** | Prompts + few-shot demonstrations | Bayesian surrogate model + generative proposal |

**Benchmark results (vs static workflows):**
- HotPotQA F1: +7.44%
- MBPP pass@1: +10%
- MATH solve rate: +10%
- GAIA overall: +up to 20%

## 3. Decisions

### D-019: EXTRACT TextGrad-style Iterative Profile Refinement — ADOPT lightweight variant
**Pattern:** After each smoke test FAIL, store a structured "what went wrong" reflection as a
`memory_type='procedural'` entry in agent-memory.db. After 3+ FAIL entries share a common tag,
Rosie triggers a profile update proposing a fix.

**Why:** Our agent profiles are static since 2026-02-17. TextGrad shows that iterative
text-based feedback loops improve agent performance 7–20%. Our smoke test already generates
FAIL signals — we just need to capture and route them.

**Lightweight implementation (no LLM overhead at gate time):**
- `smoke_test.sh` on FAIL → append structured JSON to `memory/fail-reflections.jsonl`
- Weekly: Rosie's cron reads fail-reflections and proposes profile patch
- Human approves via TODO.md review (HITL gate for profile changes)

**Effort:** LOW — 10-line addition to smoke_test.sh + weekly Rosie task

### D-020: EXTRACT HITL Checkpoint Flag — ADOPT
**Pattern:** Tasks that are "irreversible or high-risk" per AGENTS.md §12 get a `HITL_REQUIRED: true`
tag in TODO.md. Lenny flags these before Mack executes; Michael must explicitly tick them GO.

**Why:** EvoAgentX demonstrates HITL checkpoints as a first-class feature. We have the rule
in AGENTS.md but no mechanical enforcement. A simple `HITL_REQUIRED` prefix in TODO items
makes it machine-scannable and prevents Lenny from marking done without human sign-off.

**Effort:** VERY LOW — documentation + Lenny audit rule

### SKIP: Full EvoAgentX adoption
**Why:** Framework requires OpenAI API + Python 3.11 env + labeled benchmark datasets per task.
Our 4-agent cron loop doesn't have a "task dataset" to optimize against. AFlow topology search
and MIPRO Bayesian optimization are overkill at our scale. The valuable patterns are D-019 and D-020,
not the full stack.
