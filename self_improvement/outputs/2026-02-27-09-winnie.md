## 3. Ralph Loop

### Delta Since Feb 23

- **StackToHeap case study (Feb 23):** Detailed walkthrough of Ralph loops for SDK migrations — PRD decomposition → per-story Ralph runs → quality gate → commit → next story. Confirms Ralph excels at sequential, test-driven mechanical work. No new primitives introduced; pattern consolidating.
- **ralph/snarktank README update:** Post-iteration AGENTS.md update loop confirmed as official behavior. Each Ralph iteration writes discovered patterns back to AGENTS.md for next iterations and future developers. This is directly analogous to our existing fail-reflect-log D-019 hook.
- **Community consensus stable:** Ralph = atomic primitive for mechanical tasks. Not a complete architecture. No new forks or competing implementations of note.

### Assessment vs. Our Architecture

| Dimension | Ralph Loop | Our System |
|---|---|---|
| **Deterministic Execution** | ⚠️ Deterministic harness, unbounded by default | ✅ Bounded retries + escalation |
| **Verification Gates** | ✅ External check is sole gate (honest by design) | ✅ smoke_test + eval-log + cross-agent |
| **Cost/Maintenance** | 🟡 Cheap (bash) or expensive (unbounded) | 🟢 Cost-tracked, bounded |
| **Model Portability** | ✅ Model-agnostic | ✅ Explicit rotation |

### Recommendations
- **KEEP** Ralph as bounded primitive inside our system. Our D-019 fail-reflect already matches snarktank/ralph's AGENTS.md update loop.
- **TEST** (carry forward): LobeHub `/loop` quality-check tier as post-verification quality score threshold. Low priority, P4.
- **STOP** considering unbounded Ralph. Position unchanged.
