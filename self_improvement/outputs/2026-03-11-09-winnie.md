## 1. oh-my-opencode
**Overview:** Multi-agent orchestration plugin for OpenCode featuring Sisyphus, Oracle, Librarian, etc. Subagents and background tasks.
* **Deterministic execution guarantees:** Low-Medium. Relies on LLM judgment for routing and QA.
* **Verification gates:** Conducted by peer sub-agents (e.g., Momus reviews plans, Oracle reviews architecture).
* **Cost/maintenance burden:** High API cost due to multi-agent token usage and parallel execution. Maintenance is moderate.
* **Model portability:** High. Operates with Claude, Gemini, and Codex.
* **Recommendation: KEEP** - Heavily integrated into our current Oh My OpenCode v3.9.0 architecture.

## 2. Antfarm
**Overview:** TypeScript CLI tool provisioning specialized agent teams (planner, coder, tester, verifier) in OpenClaw using YAML, crons, and SQLite.
* **Deterministic execution guarantees:** High. Designed as a structured, reliable automation engine for repeatable results.
* **Verification gates:** Explicit verifier and tester agent roles are baked into the pipeline.
* **Cost/maintenance burden:** Medium. Lower API churn compared to free-form agents, but requires upfront YAML workflow definition.
* **Model portability:** Tied to OpenClaw integrations, but generally flexible within the ecosystem.
* **Recommendation: TEST** - Evaluate Antfarm's deterministic pipelines to replace some of our looser "ultrawork" bash/scripting flows.

## 3. Ralph-loop Tooling (e.g., ralph-loop-agent)
**Overview:** Iterative execution loops (Vercel AI SDK 6, OpenClaw skills) where an agent runs continuously until a stop condition is met ("crunching for hours").
* **Deterministic execution guarantees:** Low. It is exploratory and unguided until the stop condition hits.
* **Verification gates:** Purely relies on the stop conditions configured prior to execution.
* **Cost/maintenance burden:** Very High risk. Unbounded loops can quickly drain API budgets if stop conditions fail.
* **Model portability:** High (leveraging Vercel AI SDK / OpenClaw tool interfaces).
* **Recommendation: STOP** - Avoid ad-hoc Ralph-loops. The risk of runaway costs and unverified hallucinations outweighs the delegation benefits unless strictly sandboxed.
