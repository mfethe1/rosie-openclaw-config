## 1. Survey Overview

This is the most comprehensive current survey on agent memory (50+ authors, HuggingFace Daily Paper #1). It organises agent memory through **three unified lenses:**

| Lens | Question | Key concepts |
|---|---|---|
| **Forms** | What carries memory? | Token-level · Parametric · Latent |
| **Functions** | Why agents need memory? | Factual · Experiential · **Working** |
| **Dynamics** | How memory evolves? | Formation · **Evolution** · Retrieval |

**Distinction from related systems (important for our design):**
- **RAG** = retrieval at inference time from a static corpus. Our memU bridge + FTS5 qualifies.
- **Context Engineering** = what goes in the prompt window. Distinct from persistent memory.
- **Agent Memory** = memory that persists across sessions, evolves, and is agent-owned.

Our `agent-memory.db` + `memU bridge` is squarely in the Agent Memory category.

## 3. Top 3 Cherry-Pick Decisions

### D-021: Adopt Working Memory TTL (Survey §4.3)
**Pattern:** Working memory = cycle-scoped "active context" that expires automatically after use.  
**Survey finding:** Leading systems separate working memory (ephemeral, in-context) from long-term factual/experiential memory. Conflating them pollutes long-term stores with stale context.

**Implementation:**
```sql
-- Add to agent-memory.db
ALTER TABLE memories ADD expires_at TEXT DEFAULT NULL;  -- ISO timestamp
```
```python
# agent_memory_cli.py store: when --type working, default expires_at = now + 3h
# agent_memory_cli.py search: add WHERE (expires_at IS NULL OR expires_at > now())
```
**Impact:** Agents can write ephemeral scratchpad entries (current task state, mid-cycle facts) without polluting long-term search results.  
**Owner:** Mack | **Effort:** LOW (2 changes: ALTER TABLE + CLI filter)

### D-022: Adopt LightMem Sleep-Time Consolidation (Survey §5.2 + LightMem arXiv:2510.18866)
**Pattern:** Decouple expensive memory maintenance from real-time inference. Run consolidation offline (in a weekly "sleep-time" pass) that: (a) clusters similar entries, (b) merges duplicates, (c) applies quality decay to unused entries.

**LightMem key insight:** "decouples expensive memory maintenance from real-time inference, enabling reflective, high-fidelity updates without introducing latency."

**Implementation:**
```python
# New script: self_improvement/scripts/memory_consolidator.py
# Run weekly (Sunday cron alongside weekly_review.py)
# Steps:
#   1. Load all memories from agent-memory.db
#   2. Group by agent + topic (FTS5 similarity)
#   3. Merge entries where body similarity > 0.85 (keep highest provenance_score)
#   4. Apply decay: quality_score *= 0.9 for entries unused in 7+ days (use_count=0)
#   5. Archive (move to memories_archive table) entries where quality_score < 0.15
```
**Prerequisite:** Mack's D-015 quality_score column (not yet implemented) must land first.  
**Impact:** Reduces memory store bloat; surfaces high-quality entries in FTS5 ranking.  
**Owner:** Mack (quality_score prereq) then Winnie (consolidation script) | **Effort:** MEDIUM

### D-023: Adopt Hindsight Enrichment — "Reflect" Operation (Survey §5 + arXiv:2512.12818)
**Pattern:** After a task outcome is known (smoke test PASS/FAIL), retroactively enrich the memory entries created *during* that cycle with the actual outcome. This is the "reflect" operation from the Hindsight paper (retain → recall → **reflect**).

**Hindsight paper:** Three operations govern memory — retain (add), recall (access), reflect (update with outcome after known). Most systems implement retain + recall; **reflect is the missing piece**.

**Implementation:**
```bash
# In smoke_test.sh, after PASS:
python3 $CLI reflect \
  --agent $AGENT --cycle $CYCLE \
  --outcome PASS --proof $PROOF_ID \
  --quality-delta +0.1
# Adds outcome tag to all memories for this agent+cycle
# Increments use_count, adjusts quality_score upward
```
```python
# agent_memory_cli.py: new "reflect" sub-command
# UPDATE memories SET
#   outcome = ?,
#   quality_score = MIN(1.0, quality_score + delta),
#   use_count = use_count + 1
# WHERE agent_id = ? AND cycle = ?
```
**Impact:** Memories of successful task patterns get quality boosted; FTS5 ranking surfaces them more in future cycles. Closes the feedback loop currently only half-implemented via fail-reflections.  
**Owner:** Mack (agent_memory_cli.py reflect cmd) + smoke_test.sh hook | **Effort:** LOW-MEDIUM
