## 2. Antfarm (snarktank/antfarm — Feb 2026)

### Current State
- Created by **Ryan Carson** (also author of Ralph and ai-dev-tasks)
- OpenClaw-native: YAML + SQLite + cron, zero external infra
- Three bundled workflows: `feature-dev` (7 agents), `security-audit` (7 agents), `bug-fix` (6 agents)
- Built on the **Ralph loop** pattern: each agent step = fresh session, state via git + progress files
- Agents verify each other — no self-marking; verifier agent's only job is to reject incomplete work
- One-command install provisions agent workspaces, cron polling, subagent permissions
- Differentiated from Tembo (agent-agnostic enterprise) and CrewAI (Python-first)

### Scoring
| Criterion | Score | Notes |
|---|---|---|
| Deterministic execution | ★★★★★ | Same YAML steps, same order, SQLite state tracking, no LLM-driven step deviation |
| Verification gates | ★★★★★ | Explicit verifier agent per story, retry + escalation built into YAML; feedback integrity enforced by role separation |
| Cost/maintenance burden | ★★★★☆ | SQLite + cron = minimal infra; OpenClaw lock-in is the main risk; community-maintained |
| Model portability | ★★★☆☆ | Runs via OpenClaw (inherits its model support), but YAML personas are agent-agnostic; somewhat portable |

### Recommendation: **TEST — pilot on bounded feature task**
- Strongest deterministic execution + verification gate story of the three tools

## 3. Ralph-loop Pattern (snarktank/ralph + vercel-labs/ralph-loop-agent)

### Current State
- **Pattern origin:** Geoffrey Huntley's "Ralph Wiggum technique" — outer bash loop, fresh agent per iteration, state in files/git
- Two main implementations:
  - `snarktank/ralph`: bash script wrapping Amp CLI; PRD-driven (prd.json + progress.txt); 9,800+ stars
  - `vercel-labs/ralph-loop-agent`: TypeScript SDK wrapping Vercel AI SDK; `verifyCompletion` callback; `stopWhen` conditions (iteration/token/cost); 396 stars
- **Critical distinction:** True Ralph = loop control OUTSIDE the agent (bash script / SDK outer loop). Plugins that implement the loop inside the agent context do NOT solve context rot — they make it worse.
- Widely adopted pattern now absorbed into oh-my-opencode (`ralph-loop` hook) and Antfarm (core architectural principle)
- Anthropic Agent Teams (Feb 2026) does not use Ralph-loop; it uses shared task list + direct agent messaging — different tradeoff

### Scoring
| Criterion | Score | Notes |
|---|---|---|
| Deterministic execution | ★★★★★ | Outer loop control is deterministic; each iteration is bounded and isolated |
| Verification gates | ★★★☆☆ | ralph.sh: typecheck + tests per iteration; vercel SDK: custom `verifyCompletion`; no separate verifier agent |
| Cost/maintenance burden | ★★★★★ | Minimal — bash script or ~200-line TS library; no infra; open-source MIT/Apache |
| Model portability | ★★★★★ | Pattern is model-agnostic; vercel SDK supports AI Gateway (any provider) |

### Recommendation: **KEEP as foundational pattern; STOP adopting new standalone Ralph implementations**
- The pattern is already embedded in our stack via oh-my-opencode's ralph-loop hook and Antfarm's architecture
- Do NOT adopt `snarktank/ralph` standalone — it's Amp-specific and adds a dependency on a separate CLI
- `vercel-labs/ralph-loop-agent` is useful if we build custom TypeScript agent loops, but low priority given OpenClaw coverage
- Key rule to enforce: any agent loop we build must use external loop control — not in-agent continuation
- **Next action:** Add a standards note to AGENTS.md documenting "external loop control required" as an architectural constraint
