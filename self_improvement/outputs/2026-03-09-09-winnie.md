# Winnie Competitor Sweep (March 9, 2026)

## 1. Executive Summary
The state of multi-agent and agentic software engineering has aggressively shifted toward **deterministic orchestration**, **agent-native platforms**, and **systems-level safety via protocols**. The era of monolithic chat-loop wrappers is ending. New research highlights a heavy reliance on the Model Context Protocol (MCP) to decouple tools from models, the adoption of strict Finite State Machines (FSMs) or DAGs for workflows, and the implementation of multi-pass agents (like BugBot) to significantly outperform single-pass human equivalents in code review.

## 2. Framework Updates & Paradigms

### Agentic Workflows & Multi-Agent Coordination
*   **Production-Grade DAGs:** Systems like AFlow and EvoAgentX use Monte Carlo Tree Search (MCTS) to dynamically generate and optimize agentic graph topologies.
*   **FSM Orchestration (Agentic Lybic):** Complex tasks are decomposed into tiered reasoning hierarchies (Controller → Manager → Worker) governed by Finite State Machines to prevent deadlocks and ensure structured error recovery.
*   **A2A Protocol & MCP:** Google's A2A protocol and Anthropic's MCP are establishing standardized interoperability. MCP acts as the routing standard for tools, while A2A manages agent discovery and memory exchange.

### Software Engineering & Reliability (Claude Code/Manus/BugBot)
*   **End-to-End Execution:** Autonomous SWE agents (Claude Code, Manus) operate with full-repository context, handling architectural refactors rather than just snippet completion.
*   **Multi-Pass Code Review:** BugBot proves that 8 parallel review passes using randomized diff orders significantly outperforms human reviews and single-pass LLMs.
*   **Self-Healing CI/CD:** The SAPAL (Sense-Analyze-Predict-Act-Learn) framework is being used to identify flaky tests and autocorrect misconfigurations, heavily reducing pipeline bottlenecks.

### Tool Routing & Safety
*   **Strict Schemas:** MRKL-style architectures mandate typed action schemas (e.g., Pydantic/JSON) for tool arguments, eliminating ad-hoc parsing hallucinations.
*   **Verifiable Guardrails:** Safety is now middleware. "Verifier" components check policy compliance before a tool executes in an isolated environment (e.g., Firecracker VMs). 

## 3. Keep / Stop / Test Recommendations

**KEEP (Deterministic Execution & Routing)**
*   Keep our strategy of migrating crons to formal state machines (like the Ralph-loop JSON pattern we are adopting for Mack).
*   Keep moving toward strict MCP integrations for all external tooling rather than ad-hoc custom Python wrappers.

**STOP (Fragile/Monolithic Patterns)**
*   Stop relying on raw Python `subprocess` shell string generation without typed schema constraints (Pydantic/JSON Schema) for agent tool calls. We are susceptible to prompt injection and excessive agency.
*   Stop running single-pass QA checks. Adopt a multi-pass, randomized-order methodology for Lenny's verification sweeps.

**TEST (Verification Gates & Reliability)**
*   Test implementing a "Verifier Middleware" wrapper for all `execute_command` or `edit_file` tool actions that checks the action against a read-only policy before applying it.
*   Test integrating the A2A or MCP specification for inter-agent communication, moving away from file-based `shared-state.json` polling.
