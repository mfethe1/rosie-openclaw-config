**Evaluation of oh-my-opencode v3.9.0 `--worktree` flag pattern:**
Parallel feature branches inherently conflict when checked out in the same working tree. When Prometheus (planner) and Atlas (executor) or other agents operate simultaneously on different tasks in the same repository:
1. `git checkout` switches the active branch, mutating files for the other running agent.
2. LSP tools (pyright, tsserver) re-index based on the mutated state, producing false positive diagnostics.
3. Tests run by one agent fail due to incomplete code from the other agent.

**oh-my-opencode v3.9.0 pattern:**
Uses `git worktree add .git/worktrees/<task-slug> <branch>` to isolate execution.
- Disk efficient: Shares `.git` object store; no cloning required.
- Perfect isolation: Each agent has a dedicated filesystem view and dedicated LSP root path.
- Clean cleanup: `git worktree remove` easily destroys the environment post-execution.

**Decision & Implementation Plan:**
Adopt the worktree pattern for all multi-agent parallel dispatch flows in OpenClaw.
1. `dispatch_opencode.sh` will be patched to accept `--worktree <task-id>` or `WORKTREE_NAME` env var.
2. If provided, the wrapper will `git worktree add ../worktrees/$WORKTREE_NAME $BRANCH`, `cd` into it, execute the agent, and clean it up.
3. Requires tracking active worktrees in `repo_mutex.py` or similar to prevent branch collisions.
