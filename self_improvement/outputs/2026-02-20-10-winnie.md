### Key design decisions
- **Reusable test dispatch:** All execution goes through `subprocess.run` with timeout â€” same reliability model as `test_runner.py`
- **Accurate bin extraction:** Uses targeted `"bins": [...]` regex within `requires` block; ignores `config`, `install` etc. quoted strings (fixed over-capture bug during validation)
- **Optional-tool awareness:** S3 FAIL means "binary missing" â€” expected for uninstalled optional skills (slack, spotify, etc.). Informational, not a system error.
- **Memory storage:** `--store` flag writes every FAIL to `agent-memory.db` as `type=experiential` for pattern tracking
- **CI-compatible:** exit 0=all-pass, 1=any-fail; `--json` mode for programmatic parsing

# SI scripts only

# Filter to specific skill name

# JSON output + store failures to memory
python3 skill_testing_harness.py --json --store --agent winnie

# Write report to file
python3 skill_testing_harness.py --out outputs/skill-health.md
