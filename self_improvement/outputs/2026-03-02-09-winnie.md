## 3. Ralph-Loop Style Tooling (ralph-claude-code + Ralph Loop pattern)

**Primary refs:** `ralph-claude-code` v0.9.8 (tumf), Geoffrey Huntley's Ralph Loop concept

### Notable this cycle
- Ralph Loop core pattern: run same prompt repeatedly until stopping criteria; fresh context each iteration
- `ralph-claude-code` v0.9.8: Circuit breaker pattern with end detection algorithm, session management with auto-reset on anomaly
- Two-stage error filtering: JSON-aware filtering excludes field names from error detection (quality reference)
- Progress tracking via persistent JSON file (survives across iterations)
- Tests/types/lint as hard guardrails, not suggestions — same philosophy as our verification goal
- Still pre-v1.0; active development

### Assessment vs our four axes

| Axis | Ralph Loop Status |
|------|-----------------|
| Deterministic execution | HIGH — stateless fresh context each loop; no accumulated hallucination drift |
| Verification gates | HIGH — tests/lint/types must pass to continue |
| Cost/maintenance burden | LOW burden — simple pattern, minimal infra |
| Model portability | HIGH — model-agnostic by design |

### Recommendations
- **KEEP**: Ralph Loop's "persistent progress JSON + fresh context" pattern is the right model for our long-running cron jobs
- **STOP**: Running long cron tasks in a single continuous context — adopt Ralph Loop iteration model
- **TEST**: ralph-claude-code's two-stage JSON-aware error filter for our cron output parsing; directly applicable to Winnie's health sweep log analysis
