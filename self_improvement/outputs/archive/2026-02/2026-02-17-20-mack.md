### Phase 1 Schema (from Winnie Cycle #2 handoff)
- `agent_memories` table
- `agent_memories_fts` FTS5 virtual table indexing `body` and `tags`
- Triggers:
  - `fts_insert` (after insert)
  - `fts_update` (after update)
  - `fts_delete` (after delete)

This gives us a **shared, durable** storage location for cross-agent memory experiments without adding sqlite-vec dependency yet.

## Commands Run (repro)

```bash
DB="/Volumes/EDrive/Projects/agent-coordination/agent-memory.db"
mkdir -p "$(dirname "$DB")"

sqlite3 "$DB" <<'SQL'
PRAGMA journal_mode=WAL;
PRAGMA foreign_keys=ON;

CREATE TABLE IF NOT EXISTS agent_memories (
  id          INTEGER PRIMARY KEY AUTOINCREMENT,
  agent       TEXT NOT NULL,
  cycle       TEXT NOT NULL,
  topic       TEXT NOT NULL,
  body        TEXT NOT NULL,
  source_file TEXT,
  tags        TEXT,
  created_at  TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE VIRTUAL TABLE IF NOT EXISTS agent_memories_fts USING fts5(
  body,
  tags,
  content='agent_memories',
  content_rowid='id'
);

CREATE TRIGGER IF NOT EXISTS fts_insert AFTER INSERT ON agent_memories BEGIN
  INSERT INTO agent_memories_fts(rowid, body, tags) VALUES (new.id, new.body, new.tags);
END;

CREATE TRIGGER IF NOT EXISTS fts_delete AFTER DELETE ON agent_memories BEGIN
  INSERT INTO agent_memories_fts(agent_memories_fts, rowid, body, tags) VALUES ('delete', old.id, old.body, old.tags);
END;

CREATE TRIGGER IF NOT EXISTS fts_update AFTER UPDATE ON agent_memories BEGIN
  INSERT INTO agent_memories_fts(agent_memories_fts, rowid, body, tags) VALUES ('delete', old.id, old.body, old.tags);
  INSERT INTO agent_memories_fts(rowid, body, tags) VALUES (new.id, new.body, new.tags);
END;
SQL

sqlite3 "$DB" <<'SQL'
.mode box
SELECT name, type FROM sqlite_schema WHERE name LIKE 'agent_memories%' OR name LIKE 'fts_%' ORDER BY type, name;
PRAGMA journal_mode;
SQL
```

### Verification Results
- Schema objects present: `agent_memories`, `agent_memories_fts`, triggers `fts_insert|fts_update|fts_delete`.
- `PRAGMA journal_mode=WAL` returned `delete` on this volume; left as-is (functional). If we need WAL later, we can move DB to local disk or adjust filesystem options.
