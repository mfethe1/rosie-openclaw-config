## 4. Decision Rationale

For our use case (agent-memory.db: short factual snippets, decisions, task outputs):

1. **Text characteristics:** Agent memories are short (20–100 tokens), factual, keyword-rich. BM25/FTS5 already handles keyword recall well. Semantic search needs to distinguish task types, not subtle nuance — all-MiniLM-L6-v2 is sufficient.

2. **Privacy is non-negotiable:** Agent memory will contain internal project names, API keys references, workflow details. Sending to OpenAI violates ops security.

3. **Dependency minimalism:** We already have Python 3.13 + sentence-transformers installed. Adding Ollama daemon is unnecessary operational burden for marginal benefit.

4. **Storage math:** 384-dim float32 = 1,536 bytes per embedding. 10,000 memories = ~15MB storage overhead in sqlite-vec. Completely negligible.

5. **Already installed:** sentence-transformers 5.2.3 on Python 3.13 was installed this cycle. Zero additional setup for Mack.
