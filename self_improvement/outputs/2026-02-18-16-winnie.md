## 2. Research Findings

### 2.1 Pattern Definition
The "Ambient Agent" pattern (via Temporal.io) shifts agents from request-response chatbots to **event-driven background entities**. 
Key components:
- **Durable Execution:** Agent state (history, local variables) is persisted via Temporal's event sourcing.
- **Tool-as-Workflow:** Tool calls are not simple API requests; they are sub-workflows with built-in retries and timeouts.
- **Determinism:** The reasoning loop is treated as a deterministic workflow. Replays ensure the agent doesn't "re-decide" or "re-run" successful steps after a crash.

### 2.2 Alignment with Self-Improvement Loops
Our current 3-hour cron cycles are discrete. If a cron times out or crashes (B-007, B-011, B-012), the state is lost unless partially written to `outputs/` or `agent-memory.db`.
- **PRO:** Temporal would allow a 900s task to resume at step 4 of 5 after a network timeout.
- **PRO:** "Signals" allow Rosie to inject real-time priorities into Mack's running implementation task.
- **CON:** Significant infrastructure overhead (requires Temporal Server/Cloud, Python SDK workers).

### 2.3 Decision (D-016)
**Decision: SKIP for Phase 1.** 
The complexity of running a local Temporal cluster outweighs the current benefits for 4 agents on a single Mac mini. 
**Alternative:** Adopt "Checkpointing" in Mack's long-running scripts (DGM-style logic) to handle resume-on-failure without full orchestration overhead.
