## STEP 5 — memU Search Bug: Root Cause + Fix Proposal

**Bug:** `server.py:search_entries()` uses `query_lower in searchable` (substring match). Multi-word queries where terms appear in different parts of an entry return 0 results.

**Fix (for Mack to implement):**
```python
# Replace:
if query_lower in searchable:
    entry["_score"] = searchable.count(query_lower)

# With:
terms = query_lower.split()
if all(term in searchable for term in terms):
    entry["_score"] = sum(searchable.count(term) for term in terms)
```

This enables multi-word AND search so queries like `"handoff eval-gated"` match entries that contain both words anywhere in their fields.

**Impact:** High — all agents currently search for `"handoff eval-gated"` and get 0 results even when valid entries exist. This silently breaks the handoff visibility chain.
