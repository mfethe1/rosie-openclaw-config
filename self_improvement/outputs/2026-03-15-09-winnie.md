## 2. Antfarm (snarktank/antfarm — Ryan Carson)

### Current state
- **Architecture:** YAML workflow definitions + SQLite state + cron polling; zero external infrastructure
- **Bundled workflows:** feature-dev (7 agents), security-audit (7 agents), bug-fix (6 agents)
- **Agent pattern:** planner → developer → verifier → tester → reviewer; each gets fresh session, clean context
- **Key claim:** agents verify each other — developer cannot mark own work done
- **Basis:** Built on Ralph loop pattern — each step = fresh session, state in git + progress files
- **Platform lock-in:** OpenClaw-native; not agent-agnostic
- **Status:** v0.5.1 last week; actively developed; no native cost cap YAML field yet confirmed for v0.6.x

### Assessment by dimension

| Dimension | Rating | Notes |
|-----------|--------|-------|
| Deterministic execution | STRONG | YAML-defined steps run in same order every time; SQLite state prevents skip/drift; cron re-triggers if agent dies |
| Verification gates | STRONG | Structural: verifier agent exists purely to reject insufficient work; acceptance criteria in YAML per step |
| Cost/maintenance burden | LOW infra, MEDIUM ops | SQLite + cron = zero infra cost; per-run token cost per agent session adds up; no built-in cost ceiling |
| Model portability | MEDIUM | Depends on OpenClaw provider routing; inherits OMO model config if co-deployed; not independently multi-provider |

### Keep/Stop/Test
- **KEEP (pattern):** Structural verifier-as-separate-agent design — strongest verification isolation available
- **KEEP (pattern):** YAML acceptance criteria chaining — direct evidence chain, nothing lost in translation
- **TEST (priority HIGH):** Run feature-dev workflow on bounded internal task (docs generation or script cleanup); enforce $10 cost ceiling; manual checkpoint between planner and developer stages
- **TEST:** `ensure-crons` command to close idle-cron-drift gap (carried from 2026-03-14)
- **STOP:** Autonomous unattended Antfarm cron activation until native cost-cap YAML field confirmed — runaway risk estimated $3,600/mo in extreme parallel execution

- **OMO integration:** ralph-loop hook is now a first-class OMO component (hardened in v3.5.x)
- **Anthropic native:** `ralph-loop` listed as a bundled Claude Code hook in OMO — the pattern has been formally absorbed

### Assessment by dimension

| Dimension | Rating | Notes |
|-----------|--------|-------|
| Deterministic execution | STRONG | Fresh context every iteration; state in files + git; no context pollution across iterations |
| Verification gates | VARIABLE | Depends on implementation: vercel-labs/ralph has programmatic `verifyCompletion`; bash implementations rely on agent self-reporting `<promise>COMPLETE</promise>` |
| Cost/maintenance burden | LOW | Bash loop or lightweight npm; no infra; `costIs()` stop condition in vercel-labs version; primary cost is model tokens per iteration |
| Model portability | STRONG | Pattern is model-agnostic; works with Claude Code, Codex, Gemini CLI, Cursor CLI, Amp |

### Keep/Stop/Test
- **KEEP:** Ralph loop pattern as primitive inside OMO (already integrated via hook)
- **TEST (priority HIGH):** Port `smoke_test.sh` gate logic to `verifyCompletion` callback using vercel-labs/ralph-loop-agent — direct path to programmatic verification replacing agent self-report
- **TEST:** pageai-pro/ralph-loop Docker sandbox mode for overnight runs — sandboxed execution is the missing safety layer for unattended iteration
- **STOP (avoid):** Raw bash ralph loop without `costIs()` ceiling — all major implementations now include iteration/token/cost stop conditions; any internal use should adopt same
