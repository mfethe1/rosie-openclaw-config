### Refine broad exception clauses in benchmark and health gates — curiosity note: Broad exceptions mask critical infra issues like missing subprocess scripts or malformed JSON responses, making self-healing impossible.

**Root cause (not just surface error):**
- `si_benchmark_gate.py` and `winnie_health_gate.py` contained generic `except Exception as e:` blocks. In infrastructure gates, this is dangerous because it catches critical system errors (like `KeyboardInterrupt` or syntax errors) along with the expected network/IO failures, masking the true root cause of pipeline failures.

**What changed:**
- `self_improvement/scripts/si_benchmark_gate.py` — Replaced generic exception with specific `(urllib.error.URLError, json.JSONDecodeError, OSError, TimeoutError)`.
- `self_improvement/scripts/winnie_health_gate.py` — Replaced generic exception with specific `(subprocess.TimeoutExpired, subprocess.SubprocessError, json.JSONDecodeError, OSError)`.

**Value generated:** Reduces debugging time by ~15 mins per incident by surfacing exact exception types instead of masking them behind a generic fallback.

**Novel angle considered:** By enumerating exact failure domains (network vs serialization vs OS), we can programmatically route fixes (e.g., fallback to secondary network route vs disk cleanup) in future automated health checks.
