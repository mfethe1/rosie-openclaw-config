
## 2026-02-24 (Diagnostic toolkit implementation)
- Added `scripts/agent_diagnostics.py` with unified command interface:
  - `health`: checks gateway process status, port reachability, interaction-store API/LLM failure rates, structured/error logs; returns pass/fail and writes a consolidated event entry; includes exponential backoff alert state (`agent_runtime/diagnostic_state.json`).
  - `cron query`: filter by job/status/date/limit, and includes persistent failure detection for jobs with `>=3` failures in rolling 6h.
  - `cron stale`: invokes stale cleaner for running cron rows older than threshold.
  - `logs`: unified event log CLI with filters by event name, level, contains text, time range, and aliases (`errors_last_hour`, `gateway_errors_last_hour`, `errors`), JSON mode available.
  - `model status`: provider/fallback/auth chain summary from `openclaw models status --json`.
  - `model canary`: sends prompt and validates returned provider/model metadata (auth/fallback detection).
  - `model dashboard`: aggregates model usage, cron reliability, storage size, and cost hint from MEMORY context in one command.
  - `alert-state`: inspect or clear diagnostic backoff state.
- Added `scripts/diagnostics_cron.sh` nightly runner for health/stale/query/dashboard checks.
- Validation run:
  - `python3 scripts/agent_diagnostics.py health --hours 24 --json`
  - `python3 scripts/agent_diagnostics.py cron query --job cron-runtime-smoke --status failed --limit 5 --json`
  - `python3 scripts/agent_diagnostics.py cron stale --minutes 120 --json`
  - `python3 scripts/agent_diagnostics.py logs --alias errors_last_hour --json --limit 3`

  - `python3 scripts/agent_diagnostics.py model dashboard --hours 24 --json`

## 2026-02-24 (Logging infrastructure + cron jobs)
- Implemented structured logging stack in `scripts/`:
  - `logging_lib.py` (Python): writes event JSONL under `data/logs/<event>.jsonl` and mirrors to `data/logs/all.jsonl`, applies ISO timestamping, simple secret redaction, writes `event_id`.
  - `logging_lib.sh` (bash helper): `log_event <event> <level> <message> [json_data]`.
  - `logging_lib.js` (Node helper): `logEvent({ event, level, message, data, logDir })`.
  - `log_viewer.py`: CLI filters by `--event`, `--level`, `--contains`, `--since`, `--until`, supports `--json` output.
- Added nightly ingest + archival layer:
  - `log_nightly_ingest.py` parses all `data/logs/*.jsonl` into SQLite `structured_logs` table and parses raw `*.log` lines into `raw_server_logs` table.
  - dedup via row hashes/unique constraints for overlap-safe daily re-runs.
- Added daily rotation + monthly archive:
  - `log_rotate.py` rotates files over threshold (default 50MB), compresses into `data/logs/archive/`, keeps last 3 by default.
  - archives rows for `interaction`/`api` event files into monthly DB `data/logs/archives/structured_logs_YYYY-MM.sqlite`.
- Added cron runner:
  - `logging_nightly_cron.sh` executes ingest + rotate through `cron_wrapper.sh` with `AGENT_CRON_JOB=logging-nightly`, daily policy.
- Added operational doc:
  - `logging_infra_README.md` with usage, paths, and behaviors.
- Validation run:
  - `python3 -m py_compile logging_lib.py log_viewer.py log_nightly_ingest.py log_rotate.py`
  - `python3 logging_lib.py --event ... --json` (secret redaction check)
  - `python3 log_viewer.py --all --level info --limit 5 --json --path data/logs`
  - `python3 log_nightly_ingest.py --log-root ... --db ... --json` (idempotent on repeat)
  - `python3 log_rotate.py --threshold-mb 0 --keep 1 --json`
  - `bash -n logging_lib.sh logging_nightly_cron.sh`
