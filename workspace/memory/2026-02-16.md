## 22:00-08:00 ET (Feb 16-17): Comprehensive Tool Schema Fix (Google Gemini/Vertex Compatibility)

**RESOLVED**: Fixed all JSON schema validation errors blocking `google-antigravity` models.

**Root cause**: Google Gemini/Vertex API (Antigravity) rejects schemas with:
- Union types (`Type.Union([...])`) -> `image` tool used this.
- Number types in optional parameters (`Type.Optional(Type.Number(...))`) -> `sessions_list`, `browser`, and ~20 other tools used this (specifically `minimum: 1` constraint).
- **Process persistence**: The OpenClaw gateway process (PID 14663) was not restarting correctly via `openclaw gateway restart`. The LaunchAgent service restarted, but the old process held the port and kept running old code from memory, ignoring file patches.

**Solution applied**:
1. Patched 5 bundled files in `/opt/homebrew/lib/node_modules/openclaw/dist/`:
   - `pi-embedded-n26FO9Pa.js`
   - `pi-embedded-CNutRYOy.js`
   - `reply-B1AnbNl6.js`
   - `subagent-registry-kdTa9uwX.js`
   - `plugin-sdk/reply-BhWxw1_E.js`
2. Converted `Type.Union([Type.String(), Type.Array(Type.String())])` -> `Type.Array(Type.String())`.
3. Converted ALL `Type.Optional(Type.Number(...))` -> `Type.Optional(Type.String())`.
4. **Force Restart**: Manually killed the stuck gateway process (PID 14663), allowing launchd to spawn a fresh process (PID 1136/1755) that loaded the patched files.
